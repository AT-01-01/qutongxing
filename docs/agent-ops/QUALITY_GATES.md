# 质量门禁规范（强制）

## 1. 适用范围
- 所有需求（功能、修复、重构、配置变更）都必须经过本门禁。
- 未通过门禁的需求禁止进入“用户审视”与“发布候选”阶段。

## 2. 三段式流程
1. 实现轮：完成功能并通过本地基础校验。
2. 多维测试轮：按测试矩阵执行并记录结果。
3. 发布就绪轮：完成运行与风险检查，形成上线建议。

## 3. 多维测试矩阵

### 3.1 单元测试
- 范围：新增逻辑分支、复杂工具函数、关键业务规则。
- 要求：正向 + 反向 + 边界条件至少覆盖一组。

### 3.2 接口测试
- 范围：所有受影响 API。
- 要求：
  - 成功路径验证（字段完整、语义正确）
  - 失败路径验证（401、429、5xx、参数错误）
  - 契约一致性验证（前后端字段和状态码一致）

### 3.3 前后端联调测试
- 范围：涉及 UI 与接口联动的需求。
- 要求：
  - 页面可完整发起请求并正确展示
  - 鉴权、重试、错误提示行为与设计一致

### 3.4 端到端关键路径
- 默认关键路径：
  - 登录 -> 活动列表 -> 报名/管理 -> 退出登录
- 要求：链路全程可交互，无阻断错误。

### 3.5 异常与边界场景
- 至少覆盖：
  - 弱网/超时
  - 限流 429
  - 服务异常 5xx
  - 未授权 401
  - 空值/脏值输入

### 3.6 回归测试
- 要求：受影响模块全量复测，不能只测改动点。
- 重点：跨模块依赖点、历史高风险路径。

## 4. 发布就绪检查
- 构建可通过（前后端与 Android 相关链路）。
- 配置可用且安全（避免明文敏感信息直接发布）。
- 日志可观测（错误可定位、关键链路有记录）。
- 回滚方案存在且可执行（触发条件清晰）。

## 5. 阻断级缺陷定义（出现即不放行）
- 应用崩溃或核心流程不可用。
- 数据写错、状态错流转、不可恢复数据污染。
- 鉴权失效或越权访问。
- 关键接口契约破坏导致前后端无法交互。

## 6. 交付时必须附带
- 测试清单执行记录（通过/失败 + 证据）。
- 风险清单与缓解措施。
- 是否放行结论（通过 / 不通过）。
