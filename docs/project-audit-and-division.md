# 趣同行项目熟悉与分工建议

## 1. 项目现状速览

- 前端：`frontend/` 使用 Flutter，当前已经承接登录、注册、活动大厅、创建活动、活动管理、消息中心、个人中心等核心页面。
- 后端：`backend/` 使用 Spring Boot 3 + JPA + PostgreSQL，端口为 `8086`。
- 历史版本：`frontend_legacy/` 为旧 React Native 前端备份，不建议继续作为主线开发。

## 2. 当前已具备的业务能力

- 账号体系：注册、密码登录、微信/QQ/SMS 模拟登录。
- 活动主流程：活动发布、活动列表、报名、审核、删除。
- 活动增强能力：退局规则预览、退局申请、群聊、位置共享、自动打卡、成员私信开关。
- 个人中心：资料查看与编辑、活动归档、基础设置入口。

## 3. 当前主要缺口

- 前端对后端“高级能力”的接入还不完整，存在功能散落、入口不统一的问题。
- 广场页此前是静态内容，缺少真实活动发现能力。
- 管理页缺少“团长待办视角”和“参与状态追踪视角”的明确区分。
- 一些交互存在断点，例如未加入活动却能点聊天，进入后再被后端拦截。

## 4. 建议分工

### A. 产品/闭环负责人

- 梳理主链路：发现活动 -> 报名支付 -> 审核通过 -> 群聊打卡 -> 申请退出 -> 团长处理。
- 明确每个状态对应的页面入口、按钮文案、失败提示。
- 输出验收清单，保证每条链路都能走通。

### B. Flutter 前端负责人

- 负责 `frontend/lib/screens/` 下核心页面体验升级。
- 重点模块：
  - `activity_list_screen.dart`
  - `plaza_screen.dart`
  - `activity_management_screen.dart`
  - `activity_chat_screen.dart`
  - `create_activity_screen.dart`
- 目标：把后端已实现能力接进前端，并统一年轻化视觉风格。

### C. Spring Boot 后端负责人

- 负责活动状态机、审核规则、退局规则、群聊权限和异常处理。
- 重点模块：
  - `controller/ActivityController.java`
  - `controller/ActivityAdvancedController.java`
  - `service/impl/ActivityServiceImpl.java`
- 目标：保证接口语义稳定、报错明确、状态字段可支撑前端展示。

### D. 联调与测试负责人

- 建立接口联调表，逐项校验前端页面和后端字段是否一致。
- 执行回归检查：
  - 活动创建
  - 报名/取消报名
  - 审核通过/拒绝
  - 群聊
  - 定位打卡
  - 退局预览与申请
- 输出 bug 清单并推动返修。

## 5. 推荐推进顺序

1. 先完成首页、广场、活动管理的入口统一。
2. 再补齐聊天、退局、打卡等增强能力的显性入口。
3. 最后集中做视觉统一、文案统一和回归测试。
